# Asset Guide

Two assets, two jobs:

- `social_preview.png` is the 1280×640 positioning asset used as the
  GitHub Open Graph card and for first-impression social posts.
- `hero_benchmark.png` is the proof asset used inside the repo.

## Roles

- `social_preview.png`
  Use for GitHub social preview, X tweet 1, LinkedIn, Slack/Discord
  shares, and any other first-impression surface.
  Its job is to frame the pain point:
  long Claude Code sessions, less burn.

- `hero_benchmark.png`
  Use in `README.md`, benchmark-heavy threads, skeptical replies, and
  follow-up posts once the positioning is already understood.
  Its job is proof, not first impression.

Do not use `hero_benchmark.png` as the GitHub social preview.

## Messaging Rules

- Pain-first, not benchmark-first.
- Scope-first, not "wins everywhere".
- One giant prompt -> Caveman.
- Long Claude Code session under tighter limits -> Hewn.
- Expansive prose -> neutral.

## If You Regenerate `social_preview`

Keep it minimal and pain-first:

- no Caveman mention
- no benchmark bars
- no dashboard clutter
- readable at very small size

Preferred copy:

- `HEWN`
- `Long Claude Code Sessions, Less Burn`
- `Built for tighter limits, less drift, and faster turns.`

## If You Regenerate `hero_benchmark`

Keep it as a proof asset and avoid benchmark-war framing.

Preferred title direction:

- `Long Claude Code sessions — less burn, less drift, more room to work`

Preferred panel labels:

- `OUTPUT BURN`
- `USEFUL WORK PER TOKEN`
- `TIME TO COMPLETE`

Avoid titles like:

- `Hewn vs Caveman vs Verbose Claude`

The comparison can still exist inside the chart; it just should not be
the main emotional impression.
